Comandos Principais do Cypress

Alguns dos comandos mais utilizados no Cypress para automação de testes, com exemplos práticos.

  • cy.visit(): Acessa uma URL específica. 
    cy.visit('/login'); // Acessa a página de login
  • cy.get(): Seleciona um elemento com base em um seletor CSS. 
    cy.get('[data-cy=botao-submit]').click(); // Clica em um botão usando seletor customizado
  • cy.contains(): Seleciona um elemento que contém um texto específico. 
    cy.contains('Enviar').click(); // Clica em um botão que contém o texto 'Enviar'
  • cy.click(): Simula um clique em um elemento. 
    cy.get('[data-cy=botao-abrir-modal]').click(); // Clica no botão para abrir um modal
  • cy.type(): Simula a digitação de texto em campos de entrada. 
    cy.get('[data-cy=campo-usuario]').type('usuario_teste'); // Digita em um campo de texto
  • cy.clear(): Limpa o valor de um campo de entrada. 
    cy.get('[data-cy=campo-usuario]').clear(); // Limpa o campo de entrada
  • cy.select(): Seleciona uma opção em um campo select. 
    cy.get('[data-cy=dropdown-cidade]').select('São Paulo'); // Seleciona uma cidade em um dropdown
  • cy.check() e cy.uncheck(): Marca ou desmarca um checkbox ou radio button. 
    cy.get('[data-cy=aceitar-termos]').check(); // Marca uma checkbox
cy.get('[data-cy=aceitar-termos]').uncheck(); // Desmarca uma checkbox
  • cy.fixture(): Carrega dados de um arquivo JSON na pasta fixtures. 
    cy.fixture('users').then((users) => {
  cy.get('[data-cy=campo-usuario]').type(users.validUser.username);
});
  • cy.intercept(): Intercepta e simula uma resposta de uma chamada de API. 
    cy.intercept('GET', '/api/usuarios', { fixture: 'usuarios.json' }).as('getUsuarios');
cy.visit('/usuarios');
cy.wait('@getUsuarios'); // Aguarda a resposta mockada
  • cy.wait(): Pausa o teste até que um alias ou tempo específico seja alcançado. 
    cy.wait('@getUsuarios'); // Espera que a resposta da API 'getUsuarios' seja recebida
  • cy.reload(): Recarrega a página. 
    cy.reload(); // Recarrega a página atual
  • cy.viewport(): Define o tamanho da tela para simular diferentes dispositivos. 
    cy.viewport('iphone-6'); // Simula um iPhone 6
  • cy.scrollTo(): Rola a página para uma posição específica. 
    cy.scrollTo('bottom'); // Rola até o final da página
  • cy.url(): Obtém ou faz asserções sobre a URL atual. 
    cy.url().should('include', '/dashboard'); // Verifica se a URL contém '/dashboard'
  • cy.go(): Navega para frente ou para trás no histórico de navegação. 
    cy.go('back'); // Vai para a página anterior
cy.go('forward'); // Vai para a próxima página
  • cy.screenshot(): Tira uma captura de tela da página atual ou de um elemento específico. 
    cy.screenshot('pagina-login'); // Salva uma captura da página de login
  • cy.wrap(): Encapsula um objeto ou valor para permitir que Cypress interaja com ele. 
    cy.wrap({ nome: 'Cypress' }).its('nome').should('eq', 'Cypress');
  • cy.exec(): Executa um comando no terminal do sistema operacional. 
    cy.exec('npm run seed:database'); // Executa um comando para semear o banco de dados
  • cy.request(): Faz uma solicitação HTTP diretamente, útil para interagir com APIs. 
    cy.request('POST', '/api/login', {
  username: 'usuario_teste',
  password: 'senha_teste'
}).then((response) => {
  expect(response.status).to.eq(200); // Verifica se o status da resposta é 200
});
